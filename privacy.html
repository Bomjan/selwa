Privacy policy for the Selwa prototype experience.

This document explains what the current prototype collects, what remains simulated, and how future production behavior would be handled more formally.

What we collect

At the current Part B stage, Selwa primarily uses frontend-only demo flows and limited local browser storage for small interaction states such as cart count. Product catalogue data may be loaded from fallback demo content when backend services are unavailable.

Prototype data categories

TypePurposeCurrent handling
Search queriesCatalogue filteringUsed in-page and via URL parameters
Auth form inputDemo login/signup flowSent to API when available, otherwise simulated
Cart countCross-page continuityStored in local browser storage

What this means in practice

  • The live GitHub Pages version is presentation-oriented and may rely on fallback data.
  • There is no production payment processing in the current milestone.
  • Formal privacy and consent handling would be expanded in the final product phase.
